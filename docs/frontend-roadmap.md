# Frontend Roadmap

## Child and Adolescent Surface

- keep Unity as the primary surface for `child` and `adolescent`
- evolve the current bootstrap into actor-aware shells
- add age-profile presentation system with parental override
- prepare rooms/interactions only behind policy checks

## Adult Surface

- deliver parent and therapist experiences as responsive web/PWA
- keep mobile usability through the browser/PWA shell
- avoid a second native app in this phase

## Shared UI Priorities

- stronger product styling
- clearer consent and child-relationship affordances
- safer recovery/reset flows
- actor-aware navigation and policy-driven actions
